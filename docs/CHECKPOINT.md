# CHECKPOINT — Яркий пряник

Многостраничный интернет-магазин пряничного производства (эконом, до 50 товаров).
Бренд **«Яркий пряник»**, яркий стиль, статичные HTML-страницы.

## Расположение
Весь проект — в папке **`Pryanik/`**. Команды выполняем из неё.

## Как запустить (рабочий порт)
```bash
cd Pryanik && bash serve.sh 8000
```
Открыть: **http://localhost:8000** (сервер слушает `0.0.0.0`, работает проброс порта).

## Структура папок
- `*.html` — страницы, в **корне** (требование GitHub Pages: сайт с main/root).
- `assets/css/styles.css`, `assets/js/{app,products}.js` — стили и скрипты.
- `media/` — все медиа (`products/`, `video/`).
- `docs/` — вся документация (`0X-*.md`, `CHECKPOINT.md`, `CLAUDE.md`, `checklist.txt`).
- В корне также: `presentation.pdf`, `serve.sh`, `.nojekyll`.
- ⚠️ `CLAUDE.md` теперь в `docs/` — как авто-инструкции проекта в Pryanik/ больше не подхватывается.
- `script.js` в корне — осиротевший (нигде не подключён), можно удалить.

## Корзина
Состояние в `localStorage` (`yp_cart`) — общее для всех страниц; бейдж в шапке
синхронизируется. Пошаговое оформление, промокоды (`PRYANIK10`, `WB15`),
автоподсчёт доставки (бесплатно от 2000 ₽). Оплата — демо, без бэкенда.

## Проверено (Playwright)
Все 8 страниц — HTTP 200, 0 ошибок в консоли; переходы, фильтры/поиск/сортировка,
корзина до «Заказ оформлен», адаптив и мобильное меню, светлая/тёмная темы.

## Аудит по чек-листу $10k (checklist.txt)
- **Контраст WCAG AA:** пройден. `--muted` #8A7460→#6F5A47; введены `--brand-d`
  #D23F16 (кнопки/промо/badge.sale), `--mint-d` #0E8168 (badge.new/added/шаги/✓),
  `--berry-d` #D62E67 (badge.hit/cart-count). Все текстовые пары ≥4.5:1.
- **Семантика/доступность:** на всех 8 страницах — `<main id="main">` + skip-link
  «К содержимому», один `<h1>`, landmark-разметка, `:focus-visible` обводка.
- **Метатеги:** Open Graph + Twitter Card + `theme-color` на всех страницах
  (og:image = media/products/174032047.webp — временно; заменить на брендовый 1200×630).
- **Анимация:** reveal с каскадом по карточкам (stagger nth-child), уважает
  `prefers-reduced-motion`.
- **Мобайл:** отдельный drawer-паттерн, крупные тап-зоны, 0 горизонтального оверфлоу.
- **Палитра (решение):** яркая многоцветная — осознанная авторская позиция по
  прямому запросу клиента («яркие краски»), а не отход от чек-листа; цвета
  систематизированы токенами. NB: OG-картинку и телефон/почту (заглушки) заменить на реальные.

## Решения (лог)
- **Бизнес-модель (гибрид 50/50):** розница идёт через маркетплейсы (WB), сайт
  ведёт два равнозначных потока — розничный заказ И опт/B2B-заявки. На главной —
  блок «Как вам удобнее заказать» (два пути: каталог vs опт).
- **Опт — 3 направления:** мероприятия/события · корпоративные подарки с логотипом ·
  опт для бизнеса (перепродажа). Раздел в шапке переименован «Корпоративным» → «Оптом»,
  страница `corporate.html` перестроена, в форму добавлено поле «Тип заказа».

## Шрифты и мобайл (важно)
- **Шрифт:** дисплейный `Baloo 2` НЕ содержал кириллицу — все русские заголовки
  втихую откатывались на системный шрифт. Заменён на **Comfortaa** (пряничный,
  округлый, есть кириллица, OFL/бесплатно для коммерции) + тело `Nunito`. Гарнитуры
  грузятся **не блокируя рендер** (preload→onload swap) — это чинит «долго
  не включается на телефоне» (первый кадр больше не ждёт Google Fonts).
- **Reveal под `.js`:** анимации прячут контент только если JS реально загрузился
  (`<html class="js">`), + `onerror` на app.js раскрывает всё при сбое загрузки —
  контент никогда не «зависает» невидимым на медленном мобильном.
- **iOS:** все инпуты 16px (нет авто-зума при фокусе). 0px горизонтального оверфлоу.
- **Карточка товара кликабельна целиком** (растянутая ссылка `.card .card-title::after`),
  кнопка «В корзину» поверх (z-index) — во всех разделах (каталог/главная/похожие).
- **Главная:** внизу блок-форма «Оставьте заявку» (#zayavka).
- **Баг «пустой экран при переходе между разделами» (исправлен, 2 меры):**
  1. **View Transitions УДАЛЕНЫ** (`@view-transition navigation:auto` вырезан из
     styles.css) — cross-document VT оставляли залипший/пустой снапшот-оверлей при
     навигации на мобильном (особенно через бургер-меню). Переходы теперь обычные,
     надёжные; анимация появления блоков (reveal) осталась.
  2. `app.js/initReveal` дораблён: видимые во вьюпорте `.reveal` показываются сразу
     (не ждём наблюдателя) + `requestAnimationFrame` + `pageshow` (bfcache).
  3. **Cache-busting:** `styles.css/app.js/products.js` подключены с `?v=20260707b`,
     чтобы браузер гарантированно взял свежие файлы (частая причина «всё ещё пусто»).
