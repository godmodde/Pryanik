# Карта страницы и контент-структура — «Пряничный двор»

Одностраничный лендинг пряничной мастерской (Москва, доставка по России). Документ описывает порядок секций, якоря навигации, цели, ключевой контент, CTA, контент-модели и поведение при адаптиве. Полностью соответствует разметке `index.html`.

## 2. Разбор по секциям

### Шапка `header.top`
- **Назначение:** постоянная навигация и всегда доступный вход в заказ.
- **Что показываем:** логотип-«печать» (ссылка на `#top`), меню (Ассортимент, Наборы, Отзывы, Вопросы, Контакты), кнопку переключения светлой/тёмной темы, primary-кнопку «Заказать».
- **Куда ведём:** пункты меню — к соответствующим секциям; «Заказать» — сразу к форме `#kontakty`.

### 1. Герой `.hero`
- **Заголовок:** «Расписные *пряники* ручной работы».
- **Назначение:** за 5 секунд объяснить, что это и для кого.
- **Что показываем:** eyebrow «Пряничная мастерская · с 2011 года», H1, лид про медовое тесто и роспись по эскизу, приписку «Москва · доставка по России», графический орнамент.
- **Куда ведём:** основной путь — «Выбрать набор →» (`#nabory`), вторичный — «Смотреть ассортимент» (`#assortiment`).

### 2. Бегущая строка `.band`
- **Назначение:** визуальный ритм и быстрый перечень услуг.
- **Что показываем:** зацикленную ленту: Медовые пряники · Имбирные домики · Роспись глазурью · Свадебные бонбоньерки · Корпоративные подарки · Мастер-классы.
- **Куда ведём:** никуда — чисто декоративный слой (`aria-hidden`).

### 3. Ассортимент `#assortiment`
- **Заголовок:** «От пряника к чаю до пряничного дома».
- **Назначение:** раскрыть спектр продукции и услуг.

- **Куда ведём:** явного CTA нет; вовлекает и подводит к «Наборам».

### 4. Цифры `.stats`
- **Назначение:** доверие через факты.

### 5. Наборы / Цены `#nabory`

## 3. Логика навигации и повторных CTA

**Навигация:** одностраничная, все ссылки — якорные (`#assortiment`, `#nabory`, `#otzyvy`, `#faq`, `#kontakty`). Дублируется в шапке (5 пунктов) и подвале (4 пункта). Логотип в шапке и подвале возвращает к `#top`.

**Единая точка конверсии — форма `#kontakty`.** Все побуждающие к действию кнопки сходятся к ней; прямого оформления в тарифах нет — заявка обсуждается через форму/контакты.

Цепочка повторных CTA к заказу (по мере скролла):
1. **Шапка** — «Заказать» (primary) → `#kontakty`. Виден всегда.
2. **Герой** — «Выбрать набор →» (primary) → `#nabory` + «Смотреть ассортимент» → `#assortiment`. Мягкий вход: сначала знакомство.
3. **Тарифы** — по кнопке в каждой из 3 карточек → `#kontakty` («Заказать», «Заказать», «Обсудить»). Момент максимального намерения.
4. **Контакты** — «Отправить заявку →» (submit формы). Финальное действие.

Принцип: чем ниже по странице, тем «горячее» пользователь — CTA ведёт от знакомства (герой) к выбору (тарифы) и к заявке (форма).

## 4. Контент-модель

### Карточка ассортимента (`article.feat`) — 6 шт.
| Поле | Описание | Пример |
|------|----------|--------|
| Иконка | Inline-SVG, `aria-hidden` | контур сердца/домика |
| Заголовок `h3` | Название направления | «Имбирные домики» |
| Описание `p` | 1–2 фразы о сути | «Собранные и расписанные пряничные домики…» |

### Тариф (`article.plan`) — 3 шт.
| Поле | Описание | Пример |
|------|----------|--------|
| Бейдж `.badge` | Опционально, выделяет главный тариф | «Хит» (только у Подарочного) |
| Тег `.tag` | Короткая категория/повод | «К чаю» / «Подарочный» / «Для события» |
| Название `h3` | Имя набора | «Малый набор» |
| Цена `.plan-price` | Число `<b>` + единица `<small>` (или «Договорная») | «1 200 ₽» / «6 шт.» |
| Состав `ul` | Список из 5 пунктов | «6 медовых пряников», «Готовность — 2 дня» |
| Кнопка `a.btn` | Текст + ссылка (всегда `#kontakty`) | «Заказать» / «Обсудить» |

### Отзыв (`article.quote`) — 3 шт.
| Поле | Описание | Пример |
|------|----------|--------|
| Кавычка `.mk` | Декор `&ldquo;`, `aria-hidden` | „ |
| Текст `p` | Цитата клиента | «Заказывали пряники на свадьбу…» |
| Аватар `.avatar` | Инициалы, `aria-hidden` | «ЕР» |
| Имя `<b>` | Имя автора | «Елена Рощина» |
| Роль `<span>` | Контекст/статус | «невеста» / «офис-менеджер» / «мама Ани, 7 лет» |

### Вопрос FAQ (`details`) — 5 шт.
| Поле | Описание | Пример |
|------|----------|--------|
| Вопрос `summary` | Заголовок + значок «плюс» | «За сколько дней делать заказ?» |
| Ответ `.ans` | Развёрнутый текст | «Малые наборы — за 2 дня…» |

### Поле формы заявки (`form#orderForm`)
| Поле | Тип | Обязательное | Плейсхолдер |
|------|-----|--------------|-------------|
| `name` | text | да | «Имя» |
| `phone` | text | да | «+7 …» |
| `msg` | textarea | нет | «Повод, количество, пожелания по росписи» |
| — | submit | — | «Отправить заявку →» + примечание о согласии на обработку данных |

### Контакт (`ul.contact-list`) — 4 шт.
Тип (email `mailto:` / телефон `tel:` / Telegram / адрес) + подпись + маркер `.dot`.

## 5. Заметки по адаптиву

Макет — «десктоп-first» с медиазапросами `max-width`. Базовый контейнер `.wrap` фиксирован по `--maxw`, поля 24px.

| Брейкпоинт | Что меняется |
|------------|--------------|
| **940px** | Герой `.hero-grid`: с 2 колонок (`1.1fr / 0.9fr`) → 1 колонка, орнамент под текстом; зазор 48px → 36px. |
| **900px** | Ассортимент `.feat-grid`: 3 → 2 колонки (пересобираются внутренние границы ячеек). Тарифы `.price-grid`: 3 → 1 колонка, ограничение по ширине 460px и центрирование. Отзывы `.quotes`: 3 → 1 колонка. |
| **880px** | Меню `.nav-links` в шапке скрывается (`display:none`) — остаются логотип, тема, «Заказать». Контакты `.contact-grid`: 2 → 1 колонка, зазор 48px → 34px. |
| **700px** | Цифры `.stats`: 4 → 2×2; корректируются границы ячеек. |
| **600px** | Ассортимент `.feat-grid`: 2 → 1 колонка (карточки в столбик). |

**Поведение сеток:**
- **Ассортимент (6):** 3 кол. (десктоп) → 2 кол. (≤900) → 1 кол. (≤600). Сетка с общей рамкой `2px`, внутренние границы `nth-child`/`nth-last-child` пересобираются на каждом шаге, чтобы не было двойных линий по краям.
- **Тарифы (3):** 3 кол. → 1 кол. (≤900), центрированные, max-width 460px. Выделенный тариф `.plan.feature` с бейджем «Хит».
- **Цифры (4):** 4 кол. → 2×2 (≤700).
- **Герой:** 2 кол. → 1 кол. (≤940).
- **Отзывы (3) и Контакты (2):** схлопываются в одну колонку (≤900 и ≤880 соответственно).

**Прочее:** заголовки и лид используют `clamp()` (плавный масштаб без резких скачков); вертикальные отступы секций тоже на `clamp()`. Бегущая строка и орнаменты помечены `aria-hidden` и в адаптиве только уменьшаются в ритме. Блоки `.reveal` — анимация появления при скролле (не влияет на структуру).
