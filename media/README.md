# media/ — все медиафайлы проекта

Единая папка-источник для всех изображений и медиа сайта «Пряничный двор».
Правило простое: **любой медиафайл кладём сюда и отсюда же берём в код** — ничего не разбрасываем по проекту.

## Структура

| Папка | Что кладём |
|-------|-----------|
| `products/` | Фото готовых пряников для каталога (карточки товара) |
| `hero/` | Картинки для первого экрана |
| `custom/` | Примеры пряников «под заказ» (эскизы, работы по фото) |
| `corporate/` | Кейсы корпоративных заказов, логотипы клиентов |
| `reviews/` | Аватары/фото авторов отзывов |
| `logo/` | Логотип, фавикон, брендовые знаки |
| `icons/` | Пользовательские иконки (если понадобятся вместо встроенных SVG) |

## Формат и именование
- Предпочтительно **WebP** (или JPG/PNG), квадрат для карточек товара (1:1), ~1000×1000 px.
- Имена — латиницей, kebab-case, по смыслу: `medovoe-serdechko.webp`, `nabor-assorti.webp`.
- Держим оригинал и, при желании, оптимизированную версию.

## Как это подключено к каталогу
В `script.js` у товара есть необязательное поле `img`. Если оно указано — карточка
покажет фото из `media/products/`, если нет — нарисует SVG-пряник по умолчанию.

```js
{ name: "Медовое сердечко", weight: "90 г", price: 180, cat: "med",
  img: "medovoe-serdechko.webp",   // ← файл лежит в media/products/
  sym: "ck-heart", tile: "t-berry", badge: "hit" },
```

Путь в разметке собирается как `media/products/<img>`. Файлы отдаются сервером
по адресу вида `http://localhost:8000/media/products/medovoe-serdechko.webp`.

## Выгрузка с Wildberries
Реальные фото и данные товаров выгружаем из магазина на WB, фото складываем в
`products/`, а позиции прописываем в массиве `PRODUCTS` (`script.js`).
