# 04 · UI-система «Пряничный двор»

Дизайн-система строго монохромного лендинга. Все значения ниже сверены с
`styles.css` и `script.js` и соответствуют коду один в один.

---

## 1. Принцип и арт-дирекция монохрома

Проект использует **строго чёрно-белую графику без цвета и полутонов**. В палитре
нет ни одного акцентного оттенка — только «бумага» (`--paper`) и «чернила»
(`--ink`). Это отсылка к печатной графике, гравюре и прянику как оттиску формы.

Из отказа от цвета следует единственный доступный способ строить иерархию:

- **размером** — типографическая шкала от `12px` лейблов до `clamp(44px, 8vw, 96px)`
  заголовка героя;
- **начертанием** — засечный дисплейный шрифт против гротеска и моноширинного;
- **инверсией секций** — фон и текст меняются местами. Герой и ассортимент идут по
  светлой «бумаге», а бегущая строка (`.band`) и блок отзывов (`#otzyvy`) —
  сплошной «чернильный» фон с вывернутым текстом. Инверсия — главный ритмический
  приём макета.

Обводки везде толщиной `2px` (`var(--line)`), тени — жёсткие смещённые прямоугольники
без размытия (`--shadow: 7px 7px 0`), что поддерживает «оттиск/штамп» эстетику.

Служебная деталь: выделение текста тоже инвертировано —
`::selection { background: var(--ink); color: var(--paper); }`.

---

## 2. Токены (CSS custom properties)

Объявлены в `:root`. Тема переключает только четыре цветовых токена; размерные и
шрифтовые остаются едиными.

| Переменная  | Светлая тема | Тёмная тема | Назначение |
|-------------|--------------|-------------|------------|
| `--paper`   | `#FFFFFF`    | `#000000`   | Фон страницы, «бумага»; текст на инверсных секциях |
| `--ink`     | `#111111`    | `#F4F4F4`   | Основной цвет текста, обводок, заливок, «чернила» |
| `--line`    | `#111111`    | `#F4F4F4`   | Цвет всех разделителей и рамок (`2px`) |
| `--shadow`  | `7px 7px 0 var(--ink)` | `7px 7px 0 var(--ink)` | Жёсткая тень-оттиск без размытия (карточки, форма) |
| `--maxw`    | `1180px`     | `1180px`    | Максимальная ширина контентного контейнера `.wrap` |
| `--serif`   | `"Iowan Old Style", "Palatino Linotype", "Book Antiqua", Palatino, Georgia, "Times New Roman", serif` | — | Дисплейный/заголовочный шрифт |
| `--sans`    | `ui-sans-serif, system-ui, -apple-system, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif` | — | Основной текст, кнопки, поля ввода |
| `--mono`    | `ui-monospace, "SF Mono", "Cascadia Mono", "Roboto Mono", Menlo, Consolas, monospace` | — | Лейблы, надзаголовки (eyebrow), цифры, метки |

Значения `--paper` / `--ink` / `--line` заданы в трёх местах: базовый `:root`
(светлая по умолчанию), `@media (prefers-color-scheme: dark)` (системная тёмная) и
явные `:root[data-theme="light"]` / `:root[data-theme="dark"]` (ручное переключение,
имеет приоритет).

## 3. Типографика

### Роли шрифтов

- **`--serif`** — дисплей и заголовки. Все `h1, h2, h3`, цены (`.plan-price b`,
  `.stat b`), пункты бегущей строки, текст отзывов (курсив), заголовки `summary` в
  FAQ, аватары-инициалы. Начертание `font-weight: 700`, трекинг `letter-spacing:
  -0.01em`, плотная высота строки `line-height: 1.02`.
- **`--sans`** — основной читаемый текст: `body` (`line-height: 1.55`), абзацы,
  подписи авторов отзывов, кнопки, поля формы.
- **`--mono`** — служебная типографика: надзаголовки `.eyebrow`, номера `.feat .num`,
  теги тарифов `.plan .tag`/`.badge`, подписи `.stat span`, лейблы полей `.field
  label`, примечание формы `.form-note`, копирайт в футере. Характер — верхний
  регистр и широкий трекинг (`letter-spacing` до `0.24em`).

### Шкала размеров (по коду)

| Роль | Размер | Где |
|------|--------|-----|
| Заголовок героя | `clamp(44px, 8vw, 96px)`, `font-weight: 800` | `.hero h1` |
| Заголовок секции | `clamp(32px, 5vw, 54px)` | `.sec-head h2` |
| Заголовок контактов | `clamp(34px, 5vw, 58px)` | `#kontakty h2` |
| Заголовок FAQ (`summary`) | `clamp(18px, 2.4vw, 22px)` | `summary` |
| Крупная цифра статистики | `clamp(34px, 5vw, 52px)`, `800`, `tabular-nums` | `.stat b` |
| Цена тарифа | `40px`, `800` | `.plan-price b` |
| Заголовок тарифа | `27px` | `.plan h3` |
| Заголовок карточки ассортимента | `23px` | `.feat h3` |
| Лид героя | `clamp(17px, 2vw, 20px)` | `.hero p.lede` |
| Пункт бегущей строки | `18px`, курсив | `.ticker-item` |
| Основной текст секций | `15–17px` | абзацы |
| Eyebrow / лейблы | `12px`, uppercase, трекинг `0.24em` | `.eyebrow` |

Приём подчёркивания в заголовке героя: `.hero h1 u` рисует «двойную линию» через
многослойный `box-shadow: inset` из `--ink` и `--paper` — декоративное
подчёркивание без цвета.

Надзаголовок `.eyebrow` дополнен псевдоэлементом-чёрточкой (`::before` шириной `26px`,
высотой `2px`); вариант `.eyebrow.center` добавляет и `::after`.

## 4. Механика светлой/тёмной темы

Тема — это **инверсия монохрома**, а не отдельная палитра. Работают три уровня, по
возрастанию приоритета:

1. **Базовый `:root`** — светлая тема по умолчанию.
2. **`@media (prefers-color-scheme: dark)`** — если система в тёмном режиме и
   пользователь не переключал вручную, токены инвертируются автоматически.
3. **`:root[data-theme="light"]` / `:root[data-theme="dark"]`** — ручной выбор.
   Атрибут `data-theme` на `<html>` перекрывает системную настройку.

Переключатель — кнопка `.theme-toggle` (`#themeBtn`). Логика в `script.js`:

```js
var current = root.getAttribute('data-theme');
if (!current) {
  current = window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
}
root.setAttribute('data-theme', current === 'dark' ? 'light' : 'dark');
```

То есть при первом клике текущая тема вычисляется из системной настройки, затем
инвертируется установкой `data-theme`. Иконка кнопки следует за темой чисто на CSS:
`.moon` показывается при `:root[data-theme="dark"]` (и при системной тёмной, если нет
`data-theme="light"`), иначе видно солнце `.sun-i`.

Поскольку всё построено на четырёх токенах, ни один компонент не содержит
захардкоженных цветов (единичное исключение — прозрачная тень
`rgba(0,0,0,0)` у кнопки внутри выделенного тарифа), и инверсия работает глобально
без дополнительных правил.

## 5. Компоненты

### Кнопки — `.btn`, `.btn.primary`
- База: `2px` рамка `--ink`, фон `--paper`, текст `--ink`, жёсткая тень
  `3px 3px 0 var(--ink)`, `font-weight: 700`, `font-family: var(--sans)`.
- Hover: приподнимается — `transform: translate(-2px,-2px)` и увеличенная тень
  `5px 5px 0`. Active: вдавливается — `translate(1px,1px)`, тень `1px 1px 0`.
  Эффект «физической печати».
- `.primary` — инверсная (фон `--ink`, текст `--paper`); на hover инвертируется
  обратно (фон `--paper`, текст `--ink`).

### Карточки ассортимента — `.feat` в `.feat-grid`
- Сетка `repeat(3, 1fr)` в общей рамке `2px`, без зазоров (`gap: 0`); внутренние
  разделители — `border-right`/`border-bottom` на самих карточках, а крайние рамки
  гасятся селекторами `:nth-child(3n)` и `:nth-last-child(-n+3)`.
- **Инверсия по hover**: `.feat:hover { background: var(--ink); color: var(--paper); }`
  — карточка целиком выворачивается. Иконка `.ico` обведена `currentColor`, поэтому
  тоже инвертируется вместе с карточкой.
- Адаптив: `2` колонки на `≤900px`, `1` колонка на `≤600px` — с пересчётом того,
  какие рамки гасить.

### Тарифы — `.plan`, `.plan.feature`
- Сетка `repeat(3, 1fr)`, `gap: 20px`; на `≤900px` — одна колонка, `max-width: 460px`
  по центру.
- Карточка: рамка `2px`, `padding: 32px 28px`, жёсткая тень `--shadow` (`7px 7px 0`).
- `.plan.feature` — выделенный тариф, полностью инверсный (`background: var(--ink);
  color: var(--paper)`), с бейджем `.badge` в абсолютном позиционировании
  (`top: -14px; right: 24px`), вывернутым относительно карточки. Кнопка внутри
  (`.plan.feature .btn`) сделана светлой на тёмном.
- Список преимуществ `.plan li` — без маркеров, каждый пункт с псевдо-снежинкой
  `content: "\2733"` (✳) вместо буллета. Цена `.plan-price b` — засечная,
  `tabular-nums`.

### Отзывы — `.quote` (секция `#otzyvy`, инверсная)
- Вся секция инверсна: `#otzyvy { background: var(--ink); color: var(--paper); }`;
  чёрточка eyebrow здесь перекрашивается в `--paper`.
- Карточка `.quote` обведена `2px solid var(--paper)`. Крупная типографская кавычка
  `.mk` (`font-size: 60px`, засечная). Текст отзыва — засечный курсив.
- Подпись `.who`: квадратный аватар-инициал `.avatar` (рамка `--paper`, засечная
  литера), имя `--sans`, роль `--mono`.

### Аккордеон FAQ — `details / summary / .plus`
- Нативный `<details>` с нижней рамкой `2px`. `summary` со снятым маркером
  (`list-style: none` + `::-webkit-details-marker { display: none }`), засечный,
  `clamp(18px, 2.4vw, 22px)`.
- Иконка `.plus` собрана из двух псевдоэлементов-полосок (`2px`). При открытии
  `details[open] .plus::after { transform: scaleY(0); }` — вертикальная черта
  схлопывается, «плюс» превращается в «минус» с анимацией `.2s`.
- Ответ `.ans` — `--sans`, `max-width: 70ch`.

### Форма — `form.card`, `.field`
- Контейнер: рамка `2px solid var(--ink)`, тень `--shadow`, вертикальный стек полей.
- `.field` — колонка «лейбл + поле»; лейбл `--mono`, uppercase, `12px`, трекинг
  `0.1em`. Поля `input`/`textarea`: рамка `2px`, фон `--paper`, текст `--ink`,
  `--sans` `15px`; `textarea` — `resize: vertical`, `min-height: 96px`.
- Кнопка отправки растянута по центру; `.form-note` — моноширинное примечание.
- Логика (`script.js`, `#orderForm`): демо без сервера — `submit` перехватывается
  `preventDefault()`, в `.form-note` пишется «Спасибо! Заявка принята — свяжемся с
  вами сегодня.», поля теряют фокус (`blur`).

### Бегущая строка — `.band`, `.ticker`
- Инверсная полоса: `.band { background: var(--ink); color: var(--paper);
  overflow: hidden; white-space: nowrap; }`.
- Дорожка `.ticker-track` (`inline-flex`) едет анимацией `scroll-x` (`translateX(0)`
  → `translateX(-50%)`, `30s linear infinite`) — контент дублируется, поэтому
  прокрутка на половину бесшовна.
- Пункт `.ticker-item` — засечный курсив `18px`, разделён снежинкой
  `::after { content: "\2733"; }`.

### Фестончатый разделитель — `.scallop`
- Полоса высотой `18px`, залитая `--ink`, с «иконкой глазури»: полукруглые вырезы
  сделаны `mask: radial-gradient(...) repeat-x` (`mask-size: 20px 18px`), с
  `-webkit-`-дублем. Модификатор `.scallop.up` переворачивает фестоны по вертикали
  (`transform: scaleY(-1)`). Чисто декоративный элемент-«обрезка теста».

### Орнамент-медальон — `.ornament`, `.draw`, `.seal`
- Круглый SVG-медальон в герое (`.ornament svg`, до `420px`), медленно вращается:
  `animation: spin 90s linear infinite` (только при `prefers-reduced-motion:
  no-preference`).
- Эффект «рисования от руки»: элементы `.draw path/circle/ellipse` получают
  `stroke-dasharray: 1400; stroke-dashoffset: 1400` и анимацию `draw 2.4s ease
  forwards`, которая обнуляет `stroke-dashoffset` — линия прочерчивается.
- `.seal` (`34×34`) — маленькая печать-логотип рядом с брендом в шапке.

## 6. Анимация и `prefers-reduced-motion`

Движение в системе:

| Анимация | Механика | Селектор |
|----------|----------|----------|
| Рисование орнамента | `stroke-dashoffset` `1400 → 0` | `.draw …`, `@keyframes draw` |
| Вращение медальона | `rotate(360deg)`, `90s` | `.ornament svg`, `@keyframes spin` |
| Бегущая строка | `translateX(0 → -50%)`, `30s` | `.ticker-track`, `@keyframes scroll-x` |
| Появление секций | opacity/translateY при скролле | `.reveal` → `.reveal.in` |
| Плавный скролл | `scroll-behavior: smooth` | `html` |
| Микро-hover | `translate` + рост тени | `.btn`, `.theme-toggle`, `.feat` |

**Reveal при скролле** реализован на `IntersectionObserver` (`script.js`,
`threshold: 0.14`): при попадании в вьюпорт элементу добавляется класс `in`
(`opacity: 1; transform: none`), после чего наблюдение снимается (`unobserve`).

**Уважение к `prefers-reduced-motion: reduce`** — сквозное правило системы:

- `html { scroll-behavior: auto; }` — отключается плавный скролл;
- `.ornament svg` — вращение не запускается (задано только для `no-preference`);
- `.draw …` — `stroke-dashoffset: 0; animation: none` (орнамент сразу дорисован);
- `.ticker-track` — анимация только в `no-preference` (строка стоит статично);
- `.reveal` — `opacity: 1; transform: none; transition: none` (без появления);
- в `script.js` при `reduce` `IntersectionObserver` вообще не создаётся — всем
  `.reveal` сразу проставляется `in`.

## 7. Доступность

- **Видимый фокус.** Общий стиль для всех интерактивных элементов:
  `a, button, summary, input, textarea :focus-visible { outline: 3px solid
  var(--ink); outline-offset: 3px; }` — толстый контрастный контур с отступом,
  всегда монохромный и потому видимый в обеих темах.
- **Контраст.** Палитра предельно контрастна по построению: `#111111` на `#FFFFFF`
  и `#F4F4F4` на `#000000` — оба сочетания с большим запасом проходят WCAG AA/AAG.
  Инверсные секции сохраняют тот же контраст, меняя лишь направление.
- **`aria-hidden` на декоре.** Чисто декоративные элементы (орнамент-медальон,
  печать-логотип, фестончатые разделители `.scallop`, снежинки-буллеты, чёрточки
  eyebrow) не несут смысла и должны быть скрыты от скринридеров через
  `aria-hidden="true"` / `role` по месту в разметке, чтобы не засорять
  озвучивание.
- **Семантика.** FAQ построен на нативных `<details>/<summary>` — доступен и
  управляем с клавиатуры без JS. Форма использует связку `label`+`input` в `.field`.
- **Управление с клавиатуры.** Переключатель темы — обычная `<button>`; фокус,
  hover и `:active` визуально различимы.
